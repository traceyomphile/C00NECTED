1. The "amahle" Fragmentation (TCP Stream Issue)
The Problem: When you sent the name "amahle", it was split across multiple lines on the server (e.g., "amahl" and then "e"). This happened because TCP is a continuous stream of bytes, and without a defined boundary, the server's recv() just grabs whatever is immediately available in the network buffer.
The Code That Caused It (Server):
Initially, the server was either reading an arbitrary chunk of bytes or not correctly waiting for the full payload length.

# The server was blindly reading chunks without knowing the exact message size
def handle_client(client_socket, addr):
    # ... 
    while True:
        # Reading an arbitrary 1024 bytes could cut messages in half
        data = client_socket.recv(1024) 
        if not data: break
        print(f"[{username}]: {data.decode('ascii')}")


The Fixed Code (Server & Client):
We implemented Message Framing (a length-prefix protocol). The client sends the exact size of the message first, and the server reads exactly that many bytes.

# THE FIX (Client-Side Sending):
def send_framed_msg(sock, message):
    data = message.encode('ascii')
    header = f"{len(data):04d}".encode('ascii') # e.g., '0006' for 'amahle'
    sock.sendall(header + data) 

# THE FIX (Server-Side Receiving):
        while True:
            # 1. Read exactly the 4-byte length header first
            header = client_socket.recv(4) 
            if not header: break
            msg_len = int(header.decode('ascii'))

2. Port Collision ("Address already in use")

The Problem: You tried to test the requirement for concurrent clients  by opening a second terminal. The second client immediately crashed. This happened because both clients were trying to claim the exact same UDP port on your local machine for the P2P media listener.

The Code That Caused It (Client):
# Hardcoding the port means only ONE client can ever run on this machine
UDP_PORT = 6000 

def receive_udp_media():
    udp_sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
    # The second client crashed right here because 6000 was taken
    udp_sock.bind(('0.0.0.0', UDP_PORT))

The Fixed Code (Client):
We changed the port to 0, which acts as a wildcard. It tells the operating system to dynamically assign a unique, available port to every new client you open.
# THE FIX:
UDP_PORT = 0 

def receive_udp_media():
    udp_sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
    # Binding to 0 lets the OS pick a unique port for every concurrent client
    udp_sock.bind(('0.0.0.0', UDP_PORT)) 
    
    # We retrieve the assigned port so we know where to send media later
    assigned_port = udp_sock.getsockname()[1]
    print(f"[UDP] Listening for media on unique port {assigned_port}...")
